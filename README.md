**2021.09.24)**<br> 스프링 빈과 빈을 등록하는 방법 2가지 (컴포넌트 스캔과 자동 의존관계 설정, 자바 코드로 직접 스프링 빈 등록하기) 중 첫번째 방법을 공부하였다. @Component 애노테이션을 사용해서 스프링 빈으로 자동 등록을 할 수 있다. 예제에서 본 바와 같이 controller 객체를 통해서 외부 요청을 받고 service 객체를 통해서 비즈니스 로직을 만들고 repository 객체에서 데이터를 저장하는 것은 정형화된 패턴이다. 또한 controller 객체 내부에 service 객체가 있고 그 내부에는 repository 객체가 존재하기 때문에 repository 객체만 new keyword를 사용해서 생성하주고 service, controller 객체를 만들때 각 객체의 instance variable 역할을 하는 reposiroty, service 객체는 dependency injection을 통해서 삽입해 주는데, 이러한 과정을 스프링 빈으로 등록해둔다면 알아서 객체 생성 시 setter(constructor) method를 호출해서 injection을 스프링에서 알아서 해준다. 이러한 방식이 컴포넌트 스캔을 통한 자동 의존관계 설정이다. 구현 방법은 @Component 애노테이션을 사용하는데, @Controller, @Service, @Repository의 애노테이션은 전부 @Component 애노테이션을 조금 더 구체화해서 사용한 애노테이션이라고 생각하면 된다. 이후 Setter 메소드 위에 @Autowired 애노테이션을 달아준다면 알아서 Springbean에서 객체를 찾아서 dependency injection을 해준다. 또한 메인 메서드가 포함된 디렉토리의 하위 디렉토리를 스캔해서 찾아내는 방식이기 때문에 반드시 같은 디렉토리 하위에 있어야 하고, 같은 package로 감싸져 있어야 한다.<br><br>

JPA : JDBC, JDBCTemplate를 통해서 DB를 연결함으로써 DB를 연결하는것이 훨씬 쉬워졌으나, 여전히 개발자가 직접 DB를 작성해야 한다는 번거로움이 있다. JPA를 이용하면 이러한 것들을 보다 쉽게 해결할 수 있다. JPA를 사용하면 SQL과 데이터 중심의 설계에서 객체 중심의 설계로 패러다임을 전환할 수 있다. 또한 개발 생산성을 크게 높힐 수 있다. 먼저 build.gradle파일에 JPA, h2 데이터베이스 설정 추가한다. JPA는 interface다. interface만 제공을 해주고, 구현체는 hibernate를 주로 사용한다. JPA는 객체+ORM(Object Relational Mapping)이라는 기술이다. 따라서 우선 @Entity라는 애노테이션으로 맵핑을 해줄 것이다. Member class에 @Entity 애노테이션을 추가하고 instance value에 @Id @GeneratedValue 애노태이션을 추가한다. 쿼리에 아이디를 넣는 것이 아니라 DB에 값을 넣으면 DB가 아이디를 자동으로 생성해주는 것을 identity 전략이라고 한다. (strategy = GenerationType.IDENTITY) 구문을 @GeneratedValue 애노테이션 뒤에 추가해준다. 이러한 identity 전략을 사용하면 DB의 값을 쿼리를 통해서 직접 넣지 않더라고 instance들을 추가해 줄 수 있다. Repository 역시 마찬가지로, JpaMemberRepository를 만든다. 이후 MemberRepository를 implements하도록 만든 다음 methods들을 import한다. (단축키 : alt+enter -> import methods) 다음 final 형태의 EntityManager를 선언한다. (마찬가지, alt+enter -> constructor) JPA는 EntityManager라는 것으로 모두 동작을 하기 때문에 만든 것이다. 이후 bulid.gradle에서 라이브러리를 import했기 때문에 스프링부트가 자동으로 EntityManager를 만들어서 연결까지 해준다. 우린 그 만들어진 것으로부터 injection받으면 된다. 이후 save method에다가 em.persist(member); return member; 만 해주면 JPA가 insertQuerry를 다 만들어서 DB에 집어넣고 ID까지 member에다가 setID까지 전부 해준다. 마찬가지로 findById 역시 Member member = em.find(Member.class, id); return Optional.ofNullable(member)만 설정해준다. 이 다음 우선 findAll을 먼저 구현할 것인데 여기서 JPQL이라는 언어를 사용한다. 보통은 테이블을 대상으로 querry를 날렸는데 객체, 정확히는 Entity를 대상으로 querry를 날린다. return em.createQuery("select m from Member m", Member.class).getResultList(); member entity 자체를(객체 자체를) select해서 조회하면 끝난다. 기존의 SQL에서는 id name 등을 직접 찾아서 mapping해야 하지만 위와 같은 방식은 한 줄로 검색과 mapping까지 끝난다. FindByName 메소드도 구현은 비슷한데, List<Member> result = em.createQuery("select m from Member where m.name = :name", Member.class).setParameter("name", name) .getResultList(); return result.stream().findAny(); 와 같이 구현한다. JPA를 사용할때 항상 주의해야 할 것은 @Transactional이 필요하다. 따라서 서비스 계층 (MemberService)에다가 @Transactional을 우선 걸어준다. 다음 SpringConfig에 repository를 JpaMemberRepository로 바꿔주고 parameter로 Entity를 넣어야 하기 때문에 새로운 Entity를 만들고 @Autowired를 사용해서 dependency injection해준다.<br><br>
